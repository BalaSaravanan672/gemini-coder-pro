Section 1: Persona Architecture

How to inject the "Claude Code" behaviors

Option 1: Dynamic File Loading (Recommended)

Store the prompt in a Markdown file. The orchestrator reads it at startup.

Pros:
  • Easy to edit and version control.
  • Keeps orchestrator.ts clean.
  • Allows users to customize the persona without compiling.
Cons:
  • Requires a file read on every new session.
Option 2: Hardcoded String

Replace the existing short prompt with a massive multi-line string in the TS file.

Pros:
  • No external file dependencies.
  • Slightly faster startup.
Cons:
  • Makes the orchestrator file massive and hard to read.
  • Requires recompilation to change behaviors.
